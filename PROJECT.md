# 🥕 당근러닝크루 (C.R.C) 웹사이트

울산 당근러닝크루 웹사이트. **소개 페이지 + 크루원 전용 공간 + 내 정보·관리 허브**로 구성됩니다.
Firebase(Hosting + Auth + Firestore + **Storage(갤러리 사진)**) 기반의 정적 사이트입니다.
갤러리(Storage)를 쓰려면 **Blaze(종량제) 전환**이 필요하지만 사용량이 무료 할당 안이라 실제 비용은 사실상 $0 입니다 (→ 요금 섹션).
크루원 대부분이 모바일로 접속하므로 **모바일 화면을 우선**으로 만들었습니다 (데스크탑도 동일 기능).

## 한눈에 보기

| 페이지 | 대상 | 하는 일 |
|---|---|---|
| `index.html` 소개 페이지 | 누구나 | 크루 소개 · 정기런 일정 · 공식 기록 · 가입 안내/문의 |
| `app.html` 크루 공간 | 크루원 (로그인) | 가입 신청, 공지, 일정·출석체크, 투표, **갤러리(앨범·사진)**, 멤버 관리(운영진) |
| `admin.html` 내 정보·관리 | 크루원 (정보수정) / 운영진 (페이지관리·통계) | 상단 이름을 누르면 진입 — 상단 가로 책갈피 서브탭 |

**전체 흐름**: 방문자가 소개 페이지에서 크루를 알아보고 → "크루 가입하기"로 가입 신청서 작성 →
운영진이 멤버 탭에서 신청 정보를 보고 승인/거절 → 크루원이 되면 공지·일정·출첵·투표 이용.

## 페이지별 기능

### 1. 소개 페이지 (index.html)

위에서부터: 히어로(슬로건 + 크루 현황 숫자) → 소개/핵심 가치 → 정기런 일정 → 공식 기록 →
가입 안내(JOIN US) → 가입 문의.

**내용이 없는 섹션은 자동으로 숨겨집니다** (화면 + 상단/하단 메뉴 링크까지):

| 섹션 | 표시 조건 |
|---|---|
| 크루 현황 숫자 · 핵심 가치 · 정기런 일정 | 기본값(site-data.js) 또는 저장 내용이 있으면 표시. 페이지 관리에서 항목을 모두 지우고 저장하면 숨김 |
| 공식 기록 · 가입 안내 · 가입 문의 | **기본값 없음** — 페이지 관리에서 저장한 내용이 있을 때만 표시 |

- 첫 화면은 ①localStorage 캐시(마지막으로 불러온 저장 콘텐츠) → 없으면 ②site-data.js 기본값으로
  그려지고, ③Firestore 최신 내용으로 갱신+재캐시 (기본값 깜빡임 방지 — 페이지 관리 저장 시 캐시도 즉시 갱신)
- 공식 기록: 연도 탭 → 대회 카드 → 종목 탭(3km/5km/10km/하프/풀코스 — 짧은 거리부터) → `이름 | 입상 순위` 2열 표.
  순위는 자유 입력(`1위`·`우승` 등)이며 숫자로 시작하는 값이 앞으로 정렬됩니다
- 히어로의 "크루 가입하기" 버튼 → 크루 공간 가입신청 화면으로 바로 이동 (`app.html#signup`)

### 2. 크루 공간 (app.html)

**로그인/가입신청**: [로그인] [가입신청] 탭. 역할(role)에 따라 승인 대기/거절 안내 화면 또는 본화면으로 이동.

로그인하면 **화면 하단 고정 탭 5개**로 이동합니다. 탭은 눌러서, 또는 **좌/우 스와이프**로 이동합니다
(왼쪽=다음·오른쪽=이전, 세로 스크롤/가로 스크롤 영역/모달 중에는 무시).
**내 정보·관리(admin.html)는 상단의 본인 이름을 눌러** 진입하고, 거기서 오른쪽으로 밀면 크루 공간으로 복귀.

**휴대폰·브라우저 뒤로가기**를 누르면 앱이 바로 종료되지 않고 **이전 단계로** 돌아갑니다 —
열린 모달(사진 크게보기 등) → 갤러리 사진 선택/열린 앨범 → 홈이 아닌 탭은 홈으로, 순서대로 한 단계씩 닫히고,
홈 화면에서 **2초 안에 한 번 더** 누르면 앱을 나갑니다("한 번 더 누르면 나가요" 안내 표시).
(히스토리에 가드 항목을 얹어두는 방식 — `init.js` 의 `popstate` 처리)

| 하단 탭 | 기능 |
|---|---|
| 🏠 홈 | 대시보드 — 인사말 `홍길동님! + 랜덤 응원 문구`(`data/cheers.json` 40개 중 하나, 앱을 켤 때마다 바뀜), 카테고리별 가장 가까운 일정 **D-day 카드**(최대 4장, 같은 크기), **고정 공지 배너**, 오늘~2주 **다가오는 일정**, **진행 중인 투표** 미리보기. 누르면 해당 화면으로 이동: 일정 관련 → '일정' 서브탭, 공지 배너 → 소식 '공지' 필터, 투표 → 소식 '투표' 필터 |
| 📅 일정·출첵 | 서브탭 3개 — **책갈피(폴더 탭) 형태** (아래 표) |
| 📢 소식 | **공지 + 투표 통합.** 전체/공지/투표 **책갈피형 필터**. 공지: `고정(📌) \| 제목 \| 날짜` 목록, 펼쳐서 확인, 운영진 작성·고정·수정·삭제. 투표: 선택지 투표(표 변경 가능)·실시간 집계, **선택지의 인원수(`3명 60%`)를 누르면 투표자 명단 모달**, 하단에 **크루원 대비 참여율(`4/25 참여`)**. 운영진: 시작(선택지 ＋/－ 로 추가·삭제, **마감일시 지정** — 지나면 자동 마감)·수정·마감·삭제 |
| 🖼️ 갤러리 | **앨범 단위 사진첩** (→ 아래 갤러리 섹션). **카테고리 책갈피 탭**(일정 카테고리 연동 · 앨범 있는 카테고리만 노출) → 앨범 목록(표지·날짜·장수) → 앨범을 누르면 사진 그리드 → 사진을 누르면 **라이트박스**(좌우 넘기기 · 업로더·**올린날짜~원본삭제일** · **원본 저장** · 삭제). **사진 올리기**: 원본만 고르면 **미리보기에서 확인·빼기 후 [올리기]** → 브라우저가 썸네일·보기용을 자동 생성해 업로드. 앨범당 **최대 20장**. **사진 저장**: 앨범 상단 [사진 저장] → **전체/선택 다운로드**(원본, 60일 이내). 운영진: 앨범 만들기(**일정 연결 시 날짜·카테고리 자동** / 일정 없이 직접 입력)·앨범 삭제 |
| 👥 멤버 | 크루 멤버 **한 줄 목록** — `이름 \| 합류날짜` + 운영진에게만 👑 배지(멤버는 배지 없음), 운영진 화면엔 운영진 지정/해제·내보내기 버튼 추가. **멤버 줄을 누르면 멤버 정보 모달** — 크루원은 `합류·성별·연령대·대회 경력·하고 싶은 말`까지, **운영진은 여기에 계정·사는 곳·연락처(tel: 링크)가 더 보임**. 운영진 전용: **[승인 대기] [거절] [내보내기] 서브탭** — 신청 정보(성별·연령대·경력·연락처·사는 곳·하고 싶은 말) 확인 후 ✓ 승인 / 거절, 거절·내보내기 기록 보존(`role: rejected/removed`) → 복구 또는 삭제(**내보내기 삭제 = '탈퇴'** — 멤버 문서는 지우되 통계엔 탈퇴로 남김) |

**일정·출첵 서브탭**

| 서브탭 | 기능 |
|---|---|
| 📅 일정 | 일정 목록 + 카테고리 배지/필터. 카드 **왼쪽은 날짜 배지 단독**, **우측 하단 [📷 사진] + [참석] 버튼** — 사진 버튼은 갤러리 지름길(아래), 참석 버튼으로 출석체크(참석 중이면 초록색, 다시 누르면 취소), 참석자 명단(3명 초과 시 '외 x명'). 운영진: 만들기(카테고리 ＋는 **이름·색상 모달** — 원형 색상환+명도 또는 프리셋, －로 삭제, 라벨 옆 **✏️로 수정**(이름·색 재선택, 이름 변경 시 해당 일정 자동 이동) / 카테고리 칩은 각자의 색으로 표시), 카드 우측 상단 구석의 작은 ✏️ 수정·✕ 삭제 아이콘, **참석자 요약을 누르면 '참석자 관리' 모달** → 체크박스로 모든 카테고리 일정의 참석자 수정(지난 일정 포함) |

**일정 카드의 사진 버튼 (갤러리 지름길)**
- 앨범이 연결된 일정: 📷 버튼(주황 테두리) → 그 일정의 **앨범 열기**
- 앨범이 없는 일정: 📷＋ 버튼(점선) → **그 자리에서 앨범을 만들고 바로 올리기**.
  정기런 카테고리는 크루원 모두, 그 외 카테고리(대회 등)는 운영진에게만 표시
- 일정에서 만든 앨범은 **그 일정의 카테고리를 그대로 이어받아** 갤러리의 해당 카테고리 탭에 생성됨
- 일정에서 연 앨범은 **'뒤로(‹ 앨범)' 시 다시 일정 화면**으로 복귀 (갤러리 목록에서 연 앨범은 목록으로)
| 🥕 출석 현황 | **'정기런' 일정만 출첵으로 집계.** 상단에 그 달 정기런 **날짜 칩**(`7/4(토) : 5`) 한 줄 — 칩을 누르면 그 날짜의 참석자 패널이 열림(운영진은 체크박스로 수정, 다시 누르면 닫힘) + **월별 출석 매트릭스** (이름 열 고정, 좌우 스크롤, 🥕=참석, 합계), 요약 칩(정기런·총 참석수·평균 출석률, 균등 폭), ‹ › 월 이동 |
| 🏆 이달의 기록 | **월별 마일리지 보드 + 메달 랭킹** — '오늘의 마일리지'에 거리(km)를 입력하면 그 달의 누적에 더해짐. 표: `이름 \| 출석 \| 목표(km) \| 누적(km)`, 🥇🥈🥉는 **출석(정기런) 많은 순 → 누적 km 많은 순**. 본인은 ✏️로 자기 목표·누적 수정, 운영진은 전원 수정 가능, ‹ › 월 이동 (출석 현황과 달 공유) |

### 3. 내 정보·관리 (admin.html) — 상단 책갈피 허브

크루 공간 **상단의 본인 이름(`홍길동 ▾`)을 누르면** 진입합니다 (`/admin.html` 직접 접속도 가능).
화면 **상단에 책갈피 모양 가로 서브탭**이 꽂혀 있고, 역할에 따라 개수가 다릅니다:

| 책갈피 | 대상 | 내용 |
|---|---|---|
| 정보수정 | 크루원 모두 | 이름 변경 / 비밀번호 변경(현재 비밀번호 확인) / 비밀번호 힌트 등록·변경 |
| 페이지관리 | 운영진만 | 소개 페이지의 문구·공식 기록 편집 (저장 즉시 반영, 재배포 불필요) |
| 통계 | 운영진만 | 크루 데이터로 계산한 4가지 — **크루 성장**(월별 가입↑/탈퇴↓ 다이버징 막대 · 순증) · **멤버 구성**(성별·연령대·경력) · **출석률 추이**(정기런 월별) · **개근왕·출석 랭킹**(연속 출석, `YYYY년 M월~M월` 표기). 상단 **[기간 선택]**으로 시작월~끝월을 골라(기본 최근 6개월) 셋을 함께 조회. 탭을 **처음 열 때만 로드·캐시**(재방문 시 재요청 없음)하고, 출석은 기간의 **안 읽은 회차만 증분 로드**(추이·랭킹 캐시 공유)해 읽기 비용 최소화 |

허브에도 크루 공간과 같은 **하단 탭 5개**가 유지됩니다 — 누르면 `app.html#tab=…` 으로 이동해
해당 탭이 바로 열립니다. 오른쪽으로 스와이프해도 크루 공간으로 복귀합니다.
책갈피 마크업은 `#adminTabs` 의 `.admin-tab`(`data-atab="이름"`)과 `#apanel-이름` 패널 한 쌍 —
`admin/main.js` 의 위임 핸들러가 `.active` 만 바꿔 주므로 책갈피 추가는 HTML 두 줄이면 됩니다
(운영진 전용이면 `admin-only` 클래스 추가).

**[페이지관리] 책갈피**: 소개 페이지에 보이는 순서 그대로 섹션을 나열합니다. 문구류 섹션은 위에
**소개 페이지와 똑같은 미리보기**(public.css 재사용)가 있어 아래 폼에서 고치는 즉시
미리보기에 반영 → '저장'해야 실제 적용. 빈 항목이면 미리보기에 "숨겨져요" 안내가 표시됩니다.

| 섹션 (표시 순서) | 관리 내용 |
|---|---|
| 🏠 기본 | 크루 이름·메인/서브 슬로건·소개 문단·인스타 아이디 (히어로+ABOUT 미리보기) |
| 📊 크루 현황 | 히어로 아래 통계 숫자 목록 |
| 🧡 핵심 가치 | 소개 섹션의 가치 카드 (아이콘·제목·설명) |
| 🏃 정기런 | 정기런 일정 카드 (요일·시간·집결·코스·비고) |
| 🏅 공식 기록 | 대회 등록(대회명 + 연월) → 대회마다 입상 기록(이름·종목·**입상 순위**) 추가/삭제. 종목은 3km·5km·10km·하프·풀코스 순 |
| 🤝 가입 안내 · 문의 | JOIN US 절차 목록(번호 자동) + 가입 문의 문구 |

## 계정 · 로그인

- **계정 + 비밀번호** 방식만 사용 (구글·카카오 로그인, 이메일 없음).
- **계정은 아이디 형식**: 영문, 숫자, 점(.), 밑줄(_), 하이픈(-). **대소문자를 구분**합니다 (`HongGil` ≠ `honggil`).
  > 내부 동작: Firebase 인증이 이메일 형식을 요구하고 대문자를 소문자로 바꿔 저장하기 때문에,
  > 대문자는 `+소문자` 로 인코딩하고 뒤에 내부용 도메인을 붙여 저장합니다
  > (`HongGil` → `+hong+gil@crc.ulsan`). 화면에는 항상 원래 아이디만 보입니다.
  > 모바일 키보드의 자동 대문자(첫 글자)는 계정·힌트 답변 입력 칸에서 꺼두었습니다.

  > 답변이 맞을 때만 복호화되어 본인 확인이 됩니다. 답변이 쉬우면(예: 한 자리 숫자) 그만큼
  > 뚫리기 쉬우니 **답변은 남이 추측하기 어렵게** 정하도록 크루원에게 안내하세요.
  > 비밀번호를 변경하면 힌트도 자동 갱신됩니다. 힌트 없이 가입한 기존 계정은
  > 내 정보 수정에서 힌트를 등록해야 비밀번호 찾기를 쓸 수 있습니다.

## 역할(role) 체계

`members` 컬렉션 문서의 `role` 필드로 권한이 정해집니다. 아래 값(소문자)만 인식됩니다.

| role | 의미 | 권한 |
|---|---|---|
| `pending` | 승인 대기 | 가입 신청 직후 자동 부여. "승인 대기 중" 화면만 표시 |
| `member` | 일반 크루원 | 공지·일정·투표 열람, 출석체크, 투표 참여, 본인 이름·비밀번호 변경 |
| `admin` | 운영진 👑 | member 권한 전부 + 공지/일정/투표 작성·수정·삭제, 멤버 승인·거절·내보내기, 운영진 지정, 페이지 관리 접근 |
| `rejected` | 가입 거절 | "가입이 승인되지 않았어요" 화면만 표시. 거절 히스토리용 |

**흐름**: 가입 신청 → `pending` → 운영진이 멤버 탭에서 **✓ 승인** → `member` (거절 시 `rejected`,
나중에 승인으로 변경 가능) → 멤버 카드의 "운영진 지정" → `admin`

> Firebase 콘솔에서 role을 직접 수정하는 것은 **첫 운영진 1명을 지정할 때뿐**입니다.
> `Member`, `크루원` 처럼 다른 값을 넣으면 보안 규칙이 인식하지 못해 아무것도 볼 수 없게 됩니다.

## 데이터 구조 (Firestore)

| 컬렉션 | 내용 | 읽기 | 쓰기 |
|---|---|---|---|
| `members/{uid}` | 이름, 계정, role, 가입 신청 정보(성별·연락처·사는 곳·연령대·경력·하고 싶은 말) | 본인 + 크루원 | 생성: 본인(pending만) / 이름: 본인 / 그 외: 운영진 |
| `withdrawals/{uid}` | 탈퇴 기록 (가입월·탈퇴월 타임스탬프만 — 통계 전용, 개인정보 없음) | 운영진 | 운영진 |
| `notices/{id}` | 공지 (제목·내용·고정·작성자·수정일) | 크루원 | 운영진 |
| `events/{id}` | 일정 (이름·카테고리·날짜·시간·장소·메모 + `albumId` 연결 앨범) | 크루원 | 운영진 (`albumId` 만 크루원도) |
| `events/{id}/attendance/{uid}` | 출석 (참석/불참) + 러닝 기록 (`dist` km·`sec` 초) | 크루원 | 본인 + 운영진(대신 입력) |
| `albums/{id}` | 갤러리 앨범 (`title`·`date`·`category`(일정 카테고리)·`photoCount`·`coverUrl`·`eventId`) | 크루원 | 생성: 운영진 + 크루원(일정 연결만) / 수정·삭제: 운영진 (장수·표지는 크루원도) |
| `albums/{id}/photos/{pid}` | 사진 메타 (썸네일/보기/원본 URL·경로, 업로더, `createdAt`) | 크루원 | 업로드: 본인 / 삭제: 본인 + 운영진 |
| `polls/{id}` | 투표 (질문·선택지·마감 여부 + `deadline` 마감일시 `"2026-07-20T18:00"`) | 크루원 | 운영진 |
| `polls/{id}/votes/{uid}` | 표 (선택지 번호 + 이름) | 크루원 | 본인 |
| `site/content` | 소개 페이지 내용 (문구·현황·가치·정기런·가입 안내·가입 문의) | 누구나 | 운영진 |
| `site/eventCategories` | 일정 카테고리 `{ list, colors }` (기본: 대회·정기런·모임, 폼 ＋ 모달에서 이름·색 지정) | 누구나 | 운영진 |
| `mileage/{uid}` | 마일리지 (목표 km + `months` 맵에 월별 누적 km) | 크루원 | 본인 + 운영진 |
| `records/{raceId}` | 공식 기록 (대회명·연월 + `results`: 이름·종목·`rank` 입상 순위) | 누구나 | 운영진 |
| `pwhints/{계정주소}` | 비밀번호 힌트 (질문 + 암호화된 비밀번호/답변) | 단건 조회 누구나 | 본인 uid 로만 |

보안은 전부 `firestore.rules` 가 **서버 차원에서** 지킵니다 — 크루원이 아니면 공지/일정/투표를 읽을 수 없습니다.
사진 파일은 **Cloud Storage** 에 있고 `storage.rules` 가 지킵니다 (크루원만 읽기/업로드,
삭제는 본인+운영진 — 역할은 Firestore 의 members 문서를 그대로 참조).

**Storage 파일 배치** (감상용은 영구, 원본만 자동 삭제되도록 경로를 나눔 — 하이브리드):

```
gallery/live/{앨범id}/{사진id}/{thumb|display}     ← 감상용 (수명주기 규칙 밖 → 영구 보관)
gallery/expiring/{앨범id}/{사진id}/original        ← 원본 (60일 뒤 수명주기 자동 삭제)
```

## 처음 설정 (최초 1회, 약 15분)

이미 운영 중인 프로젝트(`carrotrunningcrew`)에는 필요 없고, **새 Firebase 프로젝트로 옮길 때**만 참고하세요.

1. **Firebase 프로젝트 만들기** — https://console.firebase.google.com → 프로젝트 추가
2. **웹 앱 등록** — 프로젝트 홈에서 웹(`</>`) 아이콘 → 앱 등록 → `firebaseConfig` 값 복사
3. **설정 붙여넣기** — `public/js/firebase-config.js` 의 `window.FIREBASE_CONFIG` 를 복사한 값으로 교체
4. **Authentication 켜기** — 로그인 방법에서 **이메일/비밀번호** 사용 설정
   (화면에서는 "계정"으로 보이지만 내부적으로 이 방식을 사용)
5. **Firestore 만들기** — 위치 `asia-northeast3 (서울)` 권장, 프로덕션 모드
5-1. **(갤러리를 쓰면) Storage 활성화** — Blaze 전환 + Storage 시작 + 수명주기 규칙
   (→ 🖼️ 갤러리 섹션의 '최초 활성화 체크리스트')
6. **배포** — 터미널(PowerShell)에서:
   ```bash
   npm install -g firebase-tools   # 최초 1회
   firebase login
   cd d:\Development\welcome2crc
   firebase use --add              # 프로젝트 선택
   firebase deploy                 # 규칙 + 사이트 배포
   ```
   > Git Bash 에서 `firebase login` 은 에러가 납니다 (`winpty firebase login` 으로는 가능)
7. **첫 운영진 지정** — 사이트에서 가입 신청 → 콘솔 → Firestore → `members` → 본인 문서의
   `role` 을 `admin` 으로 수정 → 새로고침

## 평소 배포 방법

| 무엇을 바꿨나 | 배포 방법 |
|---|---|
| `public/` 안의 파일 (HTML/CSS/JS) | `git push` 만 하면 GitHub Actions 가 **자동 배포** |
| **버그 수정·기능 변경을 배포** | 위와 동일하되, 설치한 사용자에게 '새 버전' 알림을 띄우려면 `public/sw.js` 의 **`VERSION` 값을 올린 뒤** push |
| `firestore.rules` (보안 규칙) | 자동 배포에 **포함되지 않음** → `firebase deploy --only firestore:rules` 직접 실행 |
| `storage.rules` (사진 보안 규칙) | 자동 배포에 **포함되지 않음** → `firebase deploy --only storage` 직접 실행 (★ 콘솔에서 Storage 활성화 후에만 가능) |
| 소개 페이지 내용 (문구·공식 기록) | 배포 불필요 — 페이지관리에서 저장하면 즉시 반영 |

- HTML/JS/CSS 에 `no-cache` 헤더 + 서비스 워커가 **네트워크 우선**이라, 온라인에서 앱을 다시 열면
  캐시를 지우지 않아도 최신이 반영됩니다. (오프라인일 때만 캐시 사용)
- 앱을 계속 켜둔 사용자에게도 업데이트를 알리려면 `sw.js` 의 `VERSION` 을 올려 배포하세요.
  → 다음에 앱을 켜거나 화면에 돌아올 때 "새 버전이 출시되었습니다!" 알림이 뜨고,
    '업데이트'를 누르면 자동으로 최신으로 새로고침됩니다.
- PR을 올리면 미리보기 주소로 자동 배포됩니다 (GitHub Actions).

갤러리(Cloud Storage)를 쓰려면 **Blaze 플랜(카드 등록)** 이 필요합니다.
Blaze 도 무료 할당량은 그대로라, **할당량을 넘긴 만큼만** 과금됩니다.

- **Firestore**: 저장 1GB, 일 5만 회 읽기 / 2만 회 쓰기 무료 — 수십 명 크루 규모로는 여유
- **Authentication**: 일반 로그인 무제한 무료
- **Hosting**: 저장 10GB, 월 360MB 전송 무료
- **Storage(사진)**: 저장 ~5GB·전송 1GB/일 무료 수준 — 갤러리 정책(원본 60일 정리 + 앨범당 20장 +
  이미지 장기 캐시 헤더) 덕에 수년간 무료 범위 안. 서서히 쌓이는 건 영구 보관하는
  보기용 사진(연 ~1GB)뿐이라 2~3년 뒤부터 **월 몇백 원** 수준
- ★ 안전장치: Google Cloud 콘솔에서 **예산 알림($1)** 을 걸어두세요 (→ 갤러리 운영 섹션)

## 알아두면 좋은 것

- **로그인 계정과 멤버 문서는 별개입니다.** 로그인 계정은 콘솔의 **Authentication → 사용자**에,
  프로필은 **Firestore → `members`** 에 있습니다. members 문서(또는 사이트의 내보내기/기록 삭제)만
  지우면 로그인 계정은 남아서 같은 계정으로 재가입 시 "이미 사용 중인 계정이에요"가 뜹니다.
  **계정 완전 삭제는 Authentication → 사용자 탭**에서 하세요.
  (같은 계정으로 재가입하면 이전 비밀번호 힌트는 자동으로 덮어써집니다)
- **`admin` 계정은 앱 운영용 시스템 계정**이라 크루원으로 취급하지 않습니다. 멤버 목록·총 인원수는 물론
  투표 참여율/표 집계·투표자 명단·출석 현황·참석자 명단·이달의 기록 어디에도 나오지 않습니다.
  (판별 기준: 계정 ID — `js/lib/account.js` 의 `SYSTEM_ACCOUNTS`. 다른 운영용 계정이 생기면 여기에 추가)
- "내보내기"한 멤버가 다시 로그인하면 자동으로 '승인 대기' 상태로 돌아옵니다.
- 크루원 본인은 자기 members 문서의 **이름(name)만** 수정할 수 있습니다 (role 변경은 규칙으로 차단).
- 투표는 표가 선택지 번호로 저장되므로, **진행 중인 투표의 선택지 수정은 오타 고치기 정도만** 안전합니다
  (순서를 바꾸거나 줄이면 기존 표가 엉킴 — 수정 모달에도 안내 표시됨).
- 운영진이 소개 페이지 콘텐츠를 저장한 적 없으면: 문구·현황·가치·정기런은 `site-data.js` 기본값이 보이고,
  공식 기록·가입 안내·가입 문의 섹션은 아예 숨겨집니다.
- 예전 갤러리(레거시 `gallery` 컬렉션)는 제거됐고, 지금 갤러리는 **새 `albums` 컬렉션 + Cloud Storage**
  를 쓰는 별개 기능입니다. 예전 데이터가 Firestore 에 남아 있으면 콘솔에서 직접 삭제하세요.

## 🖼️ 갤러리 — 정책 · 동작 · 운영

러닝 사진을 **앨범 단위**로 올리고 보는 기능. 사진 파일은 **Cloud Storage**, 앨범·사진 메타는
**Firestore**(`albums`) 에 저장됩니다. 화면 사용법은 위 '크루 공간 > 갤러리' 표 참고.

### 카테고리 · 책갈피 탭
- 앨범은 **일정 카테고리와 연동된 카테고리**를 하나 가짐 (앨범 만들 때 선택 / 일정에서 만들면 그 일정의 카테고리를 이어받음)
- 앨범 목록 위 **책갈피 탭**으로 카테고리를 전환 — **앨범이 있는 카테고리만** 탭으로 노출
- 카테고리 없는 옛 앨범은 맨 끝 **'기타'** 탭에 모임

### 앨범 만들기 (운영진 폼)
- **일정을 연결**하면 그 일정의 **날짜·카테고리가 자동 설정·잠금**되고, 저장 시 **그 일정의 앨범으로 연결**됨(`eventId` + 일정의 `albumId` 갱신) — 일정 카드 📷 버튼과 동일한 결과
- **일정 없이** 직접 제목·날짜·카테고리만 정한 단독 앨범도 가능
- 이미 앨범이 있는 일정은 드롭다운에서 제외(중복·고아 방지)

### 저장·열람 정책 — 하이브리드
- **감상(보기)은 영구** — 썸네일·디스플레이는 계속 보관해 과거 사진도 항상 볼 수 있음
- **원본 다운로드는 올린 뒤 60일(~2달)만** — 원본 파일은 **사진마다** 60일 뒤 자동 삭제
  (감상엔 영향 없음 — 라이트박스의 '원본 저장' 버튼만 "보관 기간이 지났어요"로 바뀜)
- 라이트박스에 사진마다 **올린날짜 ~ 원본 삭제 예정일**이 표시됨
- **앨범당 최대 20장**, '좋아요' 기능 없음
- 앨범 종류(대회/러닝) 구분·영구 보관(keep) 기능은 **없음** — 모든 앨범이 같은 60일 정책

### 업로드 — 원본만 고르면 브라우저가 자동 처리
사진을 고르면 바로 올라가지 않고 **미리보기**가 먼저 뜸 — 고른 사진을 확인하고
개별 **✕로 빼기**·**＋더 고르기** 후 **[올리기]**를 눌러야 업로드 시작 (20장 제한도 여기서 반영).

| 버전 | 크기 | 포맷 | 용도 | 보관 |
|---|---|---|---|---|
| 썸네일 | ~400px | WebP | 앨범 그리드 | 영구 |
| 디스플레이 | ~1600px | WebP | 라이트박스(풀스크린) | 영구 |
| 원본 | 원본 그대로 | 원본 포맷 | '원본 저장' 다운로드 | 60일 |

- **JPEG·PNG·WebP·GIF 원본은 손대지 않고 그대로** 저장 (풀 화질 + EXIF 보존).
  브라우저가 못 읽는 포맷(HEIC 등)만 고화질 JPEG 로 변환. WebP 인코딩이 안 되는
  구형 브라우저는 자동으로 JPEG 로 대체
- 아이폰 사파리는 사진 선택 시 HEIC 를 JPEG 로 자동 변환해 주므로 대부분 원본 그대로 저장됨
- 이미지에 **1년 캐시 헤더**를 붙여 재방문 시 재다운로드가 없음 (전송량 절감)
- 원본에는 다운로드용 파일명(`crc_사진id.jpg`)이 붙어 링크를 누르면 **바로 저장**됨

### 사진 저장 — 전체·선택 다운로드
- 앨범 보기 상단 **[사진 저장]** → 선택 모드(기본 **전체 선택**) → 사진을 눌러 개별 토글·**전체 선택/해제** → **[저장 (N장)]**
- 원본을 **사용자의 한 번의 동작(제스처) 안에서 연달아** 내려받아 브라우저의 **'여러 파일 다운로드'**로 처리 (별도 라이브러리·서버 압축 없음)
- 다운로드 대상은 **원본만** — **보관 기간(60일)이 지난 사진은 자동 제외**하고 안내 (감상용 썸네일·디스플레이는 다운로드 대상 아님)
- 브라우저별 차이: **Android Chrome**은 '여러 파일 다운로드 허용' 한 번 뒤 전부 저장 / **iOS Safari**는 일괄 저장이 제한적일 수 있음(한 장씩 될 수 있음). 완전 보장하려면 서버 측 ZIP 또는 Storage CORS 설정이 필요

### 자동 삭제 — 코드 없이 Storage 수명주기 규칙으로
- `gallery/expiring/` 아래 **원본만** 60일 규칙 대상 (`gallery/live/` 썸네일·디스플레이는 규칙 밖 → 영구)
- 규칙이 지우는 건 **원본 파일**뿐 — Firestore 문서·썸네일·디스플레이는 남아 감상은 영구
- '원본 저장' 버튼 노출은 사진 `createdAt` 기준 60일로 UI 가 판단

### ★ 최초 활성화 체크리스트 (콘솔에서 1회, 약 10분)

코드는 배포돼 있어도 아래를 마쳐야 갤러리가 동작합니다. 순서대로:

1. **Blaze 전환** — Firebase 콘솔 → 요금제 업그레이드 (카드 등록. 무료 할당량은 유지됨)
2. **Storage 활성화** — 콘솔 → Storage → 시작하기 (위치는 Firestore 와 같은 리전 권장)
3. **보안 규칙 배포** — 터미널에서 `firebase deploy --only storage`
   (`firestore.rules` 도 갱신됐으므로 `firebase deploy --only firestore:rules` 함께 실행)
4. **수명주기 규칙(60일 자동 삭제)** — [Google Cloud 콘솔 → Cloud Storage → 버킷](https://console.cloud.google.com/storage/browser)
   → `carrotrunningcrew.firebasestorage.app` 버킷(Firestore 와 같은 리전인 asia-northeast3 로 재생성함) → **수명 주기(Lifecycle)** 탭 → 규칙 추가:
   - 작업: **객체 삭제(Delete object)**
   - 조건: **연령(Age) 60일** + **이름 프리픽스(matchesPrefix): `gallery/expiring/`**
   > 프리픽스를 꼭 넣으세요 — 빠뜨리면 감상용(live) 썸네일·디스플레이까지 지워져 갤러리가 깨집니다.
5. **예산 알림** — Google Cloud 콘솔 → 결제 → 예산 및 알림 → 예산 $1 생성
   (넘을 일이 거의 없지만, 넘으면 메일로 알려주는 안전장치)
