# 🥕 당근러닝크루 (C.R.C) 웹사이트

울산 당근러닝크루 웹사이트. **소개 페이지 + 크루원 전용 공간 + 운영진 페이지 관리**로 구성됩니다.
Firebase(Hosting + Auth + Firestore) 기반이며, 무료 플랜(Spark)으로 운영합니다.

## 페이지 구성

| 페이지 | 대상 | 하는 일 |
|---|---|---|
| `index.html` 소개 페이지 | 누구나 | 크루 소개·정기런 일정·공식 기록·갤러리·가입 신청 |
| `app.html` 크루 공간 | 크루원 (로그인) | 공지, 일정·출석체크, 투표, 멤버 관리(운영진), 내 정보 수정 |
| `admin.html` 페이지 관리 | 운영진만 | 소개 페이지의 문구·공식 기록·갤러리 사진을 직접 수정 |

## 무엇이 달라지나 (기존 운영 방식의 문제 해결)

| 기존 방식 | 문제 | 이 사이트 |
|---|---|---|
| 크루 소개 없음 | 신규 멤버가 크루를 알 방법이 인스타뿐 | **소개 페이지** (index.html) |
| 공지/일정/투표 → 카카오 오픈채팅 | 대화에 묻혀서 다시 찾기 어려움 | **크루 공간**의 공지·일정·투표 탭 |
| 출석체크 → 당근모임 | 크루 운영과 분리된 외부 앱 | **일정마다 참석/불참 버튼** (출석 기록 저장) |
| 모집 → 인스타 DM | 신청 정보가 정리되지 않음 | **가입 신청 폼** → 운영진이 크루 공간에서 확인·승인 |
| 소개 내용 수정 → 개발자에게 요청 | 문구 하나 바꿔도 코드 수정·배포 필요 | **페이지 관리(admin.html)** 에서 저장 즉시 반영 |

## 폴더 구조

```
public/                 ← Firebase Hosting 으로 배포되는 폴더
  index.html            공개 소개 페이지 (소개·정기런·기록·갤러리·가입신청)
  app.html              크루 공간 (로그인 필요: 공지/일정·출첵/투표/멤버 + 내 정보 수정)
  admin.html            페이지 관리 (운영진 전용: 소개 문구·공식 기록·갤러리)
  css/style.css         기본 디자인 테마
  css/v2.css            크루 공간 스타일
  css/v3.css            기록 종목 탭·갤러리 앨범·모달·관리자 페이지 스타일
  js/site-data.js       소개 페이지 기본값 (관리자 페이지 저장 내용이 우선)
  js/firebase-config.js ★ Firebase 연결 설정 (아래 처음 설정 3번에서 붙여넣기)
  js/public.js          소개 페이지 동작 (Firestore 콘텐츠 자동 로드)
  js/app.js             크루 공간 동작
  js/admin.js           페이지 관리 동작
  images/               크루 사진 (site-data.js 기본값용)
firebase.json           Firebase 배포 설정 (캐시 헤더 포함)
firestore.rules         DB 보안 규칙 (누가 뭘 읽고 쓸 수 있는지)
firestore.indexes.json  Firestore 인덱스 설정
.github/workflows/      master 푸시 시 Hosting 자동 배포 (GitHub Actions)
```

## 처음 설정 (최초 1회, 약 15분)

### 1. Firebase 프로젝트 만들기
1. https://console.firebase.google.com 접속 (Google 계정 필요)
2. **프로젝트 추가** → 이름 예: `crc-web` → (애널리틱스는 꺼도 됨) → 만들기

### 2. 웹 앱 등록
1. 프로젝트 홈에서 **웹(`</>`)** 아이콘 클릭
2. 앱 닉네임 예: `crc-site` → 앱 등록 (Hosting 체크는 안 해도 됨)
3. 화면에 나오는 `firebaseConfig = { apiKey: "AIza..." ... }` 값 복사

### 3. 설정 붙여넣기
`public/js/firebase-config.js` 를 열어 복사한 값으로 교체:

```js
window.FIREBASE_CONFIG = {
  apiKey: "AIza................",
  authDomain: "crc-web.firebaseapp.com",
  projectId: "crc-web",
  storageBucket: "crc-web.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abcdef",
};
```

### 4. 로그인 기능 켜기 (Authentication)
1. 왼쪽 메뉴 **제품 카테고리 → 보안 → Authentication** → 시작하기
   - 콘솔 버전에 따라 "빌드"/"구축" 카테고리에 있을 수도 있습니다.
   - 안 보이면: 사이드바의 **"모든 제품"** → Authentication 선택 (한 번 열면 사이드바에 고정됨)
2. **로그인 방법(Sign-in method)** 탭에서:
   - **이메일/비밀번호** → 사용 설정 → 저장
   - (사이트 로그인은 이메일/비밀번호 방식만 사용합니다)

### 5. 데이터베이스 만들기 (Firestore)
1. 왼쪽 메뉴(또는 "모든 제품")에서 **Cloud Firestore** → 데이터베이스 만들기
2. 위치: `asia-northeast3 (서울)` 권장 → **프로덕션 모드**로 시작
   (보안 규칙은 다음 단계에서 우리 파일로 덮어씁니다)

### 6. 배포 (Firebase CLI)
터미널에서:

```bash
npm install -g firebase-tools   # 최초 1회
firebase login                  # 브라우저로 Google 로그인
cd d:\Development\welcome2crc
firebase use --add              # 위에서 만든 프로젝트 선택, 별칭은 default
firebase deploy                 # 보안 규칙 + 사이트 배포
```

> ⚠️ **터미널은 PowerShell(또는 CMD)을 사용하세요.** Git Bash에서 `firebase login` 하면
> `Cannot run login in non-interactive mode` 에러가 납니다.
> (Git Bash를 꼭 쓰려면 `winpty firebase login` 처럼 앞에 `winpty`를 붙이면 됩니다.)

완료되면 `https://프로젝트ID.web.app` 주소가 나옵니다. 🎉

### 7. 첫 운영진(관리자) 지정
1. 배포된 사이트의 **크루 공간**에서 본인 계정으로 가입 (이메일/비밀번호)
2. Firebase 콘솔 → **Firestore Database** → `members` 컬렉션 → 본인 문서 클릭
3. `role` 필드 값을 `pending` → **`admin`** 으로 수정
4. 사이트 새로고침 → 👑 운영진 메뉴(공지 작성, 승인 등)가 나타납니다

이후 다른 멤버는 사이트 안에서 승인/운영진 지정이 가능하니, 콘솔에 다시 들어갈 일은 거의 없습니다.

## 평소 배포 방법

| 무엇을 바꿨나 | 배포 방법 |
|---|---|
| `public/` 안의 파일 (HTML/CSS/JS) | `git push` 만 하면 GitHub Actions 가 **자동 배포** |
| `firestore.rules` (보안 규칙) | 자동 배포에 **포함되지 않음** → `firebase deploy --only firestore:rules` 직접 실행 |
| 소개 페이지 내용 (문구·기록·사진) | 배포 불필요 — **페이지 관리(admin.html)** 에서 저장하면 즉시 반영 |

- HTML/JS/CSS 에 `no-cache` 헤더가 설정되어 있어 배포 직후 새로고침만 하면 최신 버전이 보입니다.
  (혹시 이전 화면이 보이면 한 번만 강력 새로고침: PC `Ctrl+F5`, 모바일은 브라우저 캐시 삭제)

## 역할(role) 체계

멤버마다 Firestore `members` 컬렉션 문서의 `role` 필드로 권한이 정해집니다.
값은 정확히 아래 세 단어(소문자)만 인식됩니다.

| role | 의미 | 권한 |
|---|---|---|
| `pending` | 승인 대기 | 회원가입 직후 자동 부여. 크루 공간 이용 불가, "승인 대기 중" 화면만 표시 |
| `member` | 일반 크루원 | 공지·일정·투표 열람, 출석체크, 투표 참여, 본인 이름·비밀번호 변경 |
| `admin` | 운영진 👑 | member 권한 전부 + 공지/일정/투표 작성·삭제, 멤버 승인·거절·내보내기, 운영진 지정, **페이지 관리(admin.html)** 접근 |
| `rejected` | 가입 거절 | 크루 공간 이용 불가, "가입이 승인되지 않았어요" 화면만 표시. 거절 히스토리용 |

**흐름: `pending` → `member` → `admin` (거절 시 `rejected`)**

1. 크루 공간에서 회원가입하면 자동으로 `pending`
2. 운영진이 크루 공간 → **멤버 탭 → 승인 대기 탭 → ✓ 승인** 을 누르면 `member`,
   **거절** 을 누르면 `rejected` (거절 탭에 기록이 남고, 나중에 승인으로 변경 가능)
3. 운영진이 멤버 카드의 **"운영진 지정"** 을 누르면 `admin` (해제도 가능)

> Firebase 콘솔에서 role을 직접 수정하는 것은 **첫 운영진 1명을 지정할 때뿐**입니다
> (위 "처음 설정 7번" 참고). 이후 승인/운영진 지정은 전부 사이트 안에서 처리하세요.
> `Member`, `크루원` 처럼 다른 값을 넣으면 보안 규칙이 인식하지 못해 아무것도 볼 수 없게 됩니다.

## 사용 흐름

- **일반 방문자**: 소개 페이지 → 가입 신청서 제출 (또는 인스타 DM)
- **운영진**: 크루 공간 → 멤버 탭에서 신청서 확인 → 게스트런 안내 → 크루 공간 가입 유도 → 승인
- **크루원**: 크루 공간에서 공지 확인, 일정에 참석/불참 체크, 투표 참여
- **소개 페이지 내용 수정**: 운영진이 페이지 관리(⚙️)에서 문구·기록·사진 수정 → 저장 즉시 반영

## 로그인 · 계정 관리

- 로그인은 **계정(이메일) + 비밀번호** 방식만 사용합니다 (구글·카카오 로그인 없음).
- **회원가입**: 계정(이메일) · 이름 · 비밀번호 입력 → 운영진 승인 후 이용
- **내 정보 수정**: 로그인 후 상단의 **본인 이름을 클릭** → 이름 변경, 비밀번호 변경(현재 비밀번호 확인 후)
- **비밀번호를 잊은 경우**: 로그인 화면 → "비밀번호를 잊었나요?" → 계정(이메일) 입력 →
  받은 메일의 링크에서 새 비밀번호 설정 → 새 비밀번호로 로그인
  > 본인 확인 없이 화면에서 바로 새 비밀번호를 정하게 하는 방식은
  > 누구나 남의 계정을 탈취할 수 있어 지원하지 않습니다. (메일 링크 확인이 필수)

## 페이지 관리 (admin.html) — 운영진 전용

크루 공간 헤더의 **⚙️ 아이콘**(또는 `/admin.html` 직접 접속)으로 들어갑니다.
운영진(`role: admin`) 계정만 접근 가능하며, 저장 내용은 Firestore 에 저장되어
소개 페이지가 열릴 때 자동으로 반영됩니다. (재배포 불필요)

| 탭 | 관리 내용 |
|---|---|
| 📝 소개 문구 | 크루 이름·슬로건·소개 문단·인스타 아이디 / 크루 현황 숫자 / 핵심 가치 카드 / 정기런 일정 |
| 🏅 공식 기록 | 대회 등록(대회명 + 연월) → 대회마다 종목(풀코스·하프·10km·5km)별 기록 추가/삭제 |
| 📸 갤러리 | 앨범 만들기 → 사진 업로드(자동 압축)·삭제. 소개 페이지에서 앨범을 누르면 사진 뷰어가 열립니다 |

- 아직 관리자 페이지에서 저장한 적이 없는 항목은 `js/site-data.js` 의 기본값이 표시됩니다.
- 사진은 Firebase Storage 없이(무료 플랜 유지) 자동 압축 후 Firestore 에 저장됩니다.
  한 장당 약 0.3~0.8MB 로 저장되므로 앨범당 수십 장 규모에 적합합니다.

## 로컬 미리보기

모듈(JS import)을 사용하므로 파일을 직접 열면 안 되고, 간단한 서버가 필요합니다:

```bash
cd d:\Development\welcome2crc
npx serve public
# 또는 firebase 로그인이 되어 있다면: firebase serve
```

→ http://localhost:3000 (serve 기준)

## 콘텐츠 수정

- **가장 쉬운 방법: 관리자 페이지(`admin.html`)에서 수정** — 저장 즉시 반영, 코드/배포 불필요
- `public/js/site-data.js` 는 관리자 페이지에서 저장한 적 없을 때 보이는 **기본값**입니다

## 요금

Firebase 무료 플랜(Spark)으로 충분합니다:
- Firestore: 저장 1GB / 일 5만 회 읽기 / 2만 회 쓰기 무료 — 수십 명 크루 규모로는 여유
  (갤러리 사진도 Firestore 에 압축 저장되므로 대략 1,500~3,000장까지 무료 범위)
- Authentication: 무제한 무료 (일반 로그인)
- Hosting: 10GB 저장 / 월 360MB 전송 무료

## 알아두면 좋은 것

- 보안은 `firestore.rules` 가 지킵니다. 크루원이 아니면 공지/일정/투표를 **서버 차원에서** 읽을 수 없습니다.
  소개 페이지 콘텐츠(문구·기록·갤러리)는 누구나 읽을 수 있지만 **쓰기는 운영진만** 가능합니다.
- 크루원 본인은 자기 문서의 **이름(name)만** 수정할 수 있습니다 (role 변경은 운영진만 — 규칙으로 차단).
- "내보내기"한 멤버가 다시 로그인하면 자동으로 '승인 대기' 상태가 됩니다 (계정 자체 삭제는 콘솔 → Authentication 에서).

  크루 공식 기록(대회별 종목 기록), 갤러리(앨범·사진)를 운영진이 직접 수정 → 저장 즉시 소개 페이지에 반영
- **공식 기록 개편**: 연도 탭 → 대회 카드 → 대회 안 **종목(풀코스/하프/10km/5km) 탭** + 기록순 정렬
- **갤러리 개편**: 앨범 단위(커버·이름·장수) 표시, 앨범 클릭 시 전체화면 사진 뷰어.
  사진은 브라우저에서 자동 압축 후 Firestore 저장 (Storage 불필요)
- **로그인 개편**: 구글 로그인 제거 → 계정(이메일)/비밀번호 전용.
  가입 폼 순서 정리(계정 → 이름 → 비밀번호), 비밀번호 재설정 모달(메일 링크 방식) 추가
- **내 정보 수정**: 크루 공간 상단 이름 클릭 → 이름·비밀번호 변경 모달
- **헤더 정리**: 페이지 관리 버튼 ⚙️ 아이콘화, 모바일에서 한 줄 유지(이름 말줄임)
- **보안 규칙**: 공개 콘텐츠 컬렉션(site/records/gallery) 추가, 본인 name 필드만 수정 허용 규칙 추가
  → 이 버전 배포 시 `firebase deploy --only firestore:rules` 1회 필요
- **캐시 설정**: HTML/JS/CSS `no-cache` 헤더 추가 (배포 후 이전 버전이 보이는 문제 방지)

### 최초 버전
- 소개 페이지(index.html) + 크루 공간(app.html): 공지 / 일정·출석체크 / 투표 / 멤버 관리(승인·운영진 지정)
- 가입 신청 폼 → Firestore 저장 → 운영진이 크루 공간에서 확인
